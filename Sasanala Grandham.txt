Variable Naming:
NEVER LEAVE VAR NAMES WITH VAGUE NAMES: {CAN CHANGE LATER}
-> Global variables - ALL_CAPS
-> Function name - all_smalls_with_underscore
-> Local variables - camelCaseNaming
-> Parameters - PascalCaseNaming
-> Leftovers - Capital_with_underscore
-> Ex: if var name is count then make it count_
    local and param - countL / countP


Coding:
-> Strictly no input in function. If needed then either parameters or setters and getters
-> Break things into different files
-> Make code generic
-> Never take input in modules
-> Do not call any function in the module files. Only in main
